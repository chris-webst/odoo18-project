Odoo 18 Enterprise — Instalační průvodce

Nejdříve — co vlastně instalujeme
?
Odoo vs. databáze — klíčový rozdíl
Přečti si před začátkem, ušetří ti to zmatek
▶

Spousta lidí si myslí, že Odoo a databáze je totéž. Není. Jsou to dva samostatné programy, které spolupracují.

Odoo = webová aplikace (frontend)
To, co vidíš v prohlížeči — tlačítka, formuláře, reporty, grafy. Odoo přijímá tvoje akce, zpracovává logiku a zobrazuje výsledky. Samo o sobě ale nic neukládá.
PostgreSQL = zálohovací systém (backend)
Program, který ukládá všechna data na disk — zákazníky, faktury, produkty, pohyby skladu. Je to trezor na data bez vlastního rozhraní. Odoo do něj zapisuje a čte z něj.

Proč je to důležité? Když něco nefunguje, musíme vědět kde hledat — jestli je problém ve frontend vrstvě (Odoo — zobrazení, logika) nebo v backend úložišti (PostgreSQL — data, připojení). Docker spustí oba programy najednou a propojí je.

ShrnutíOdoo = co vidíš. PostgreSQL = kde to žije. Docker = kdo to spouští.
Fáze 1 — Software
1
Docker Desktop — stažení a instalace
Program, který spouští Odoo a databázi izolovaně
▶

Co je Docker? Docker umí na tvém počítači spustit izolované "minipočítače" zvané kontejnery. Jeden bude Odoo, druhý databáze PostgreSQL. Nemusíš nic složitě instalovat — vše je zabalené uvnitř kontejnerů a systém zůstane čistý.

Stažení: Jdi na docker.com/products/docker-desktop. Nejdříve zjisti, jaký chip máš — klikni na logo Apple v systémové liště macOS (černá lišta úplně nahoře na obrazovce, ne lišta samotné aplikace) → "O tomto Macu". Pokud vidíš "Apple M1/M2/M3/M4", stáhni Mac with Apple chip. Pokud "Intel", stáhni Mac with Intel chip.

Instalace: Otevři stažený soubor .dmg a přetáhni Docker do složky Aplikace. Spusť Docker z Launchpadu. Při prvním spuštění tě macOS požádá o heslo — zadej heslo ke svému uživatelskému účtu na tomto zařízení (heslo, které používáš k přihlášení do Macu, nikoliv Apple ID).

Jak poznáš, že Docker běží: V systémové liště macOS (černá lišta úplně nahoře) se objeví malá ikonka velryby. Klikni na ni — pokud vidíš "Docker Desktop is running", vše je v pořádku. Pokud je u ikonky vykřičník, Docker ještě startuje nebo čeká na potvrzení podmínek — klikni na ikonku velryby a otevři okno Docker Desktop, kde uvidíš popis problému.

Hotovo, když: Ikonka velryby v systémové liště macOS je bez vykřičníku a kliknutím zobrazí "Docker Desktop is running".
2
Cursor — stažení, instalace a přihlášení
Editor kódu s AI — pracujeme tu celou dobu
▶

Co je Cursor? Editor kódu s vestavěnou AI. Budeme ho používat jako centrální místo — odsud spouštíme příkazy v terminálu, upravujeme konfigurační soubory a ptáme se AI na problémy.

Stažení a instalace: Jdi na cursor.com, klikni Download a stáhni verzi pro macOS. Otevři .dmg a přetáhni do Aplikací.

Přihlášení: Po spuštění Cursor zobrazí přihlašovací obrazovku — přihlas se přes GitHub (tlačítko "Continue with GitHub"). Pokud GitHub účet nemáš, vytvoř si ho zdarma na github.com — budeš ho potřebovat i pro stažení Enterprise kódu Odoo.

Nastavení rozložení panelů: Průzkumník souborů může být na pravé straně. Klikni na něj pravým tlačítkem a vyber "Move Primary Side Bar to Left". AI chat otevřeš zkratkou Cmd+L a přetáhneš ho na pravou stranu.

Hotovo, když: Cursor je otevřený, jsi přihlášen přes GitHub a průzkumník souborů je v levém panelu.
3
Rozšíření a automatické formátování
Docker extension, Python, Format on Save
▶

Instalace rozšíření: Stiskni Cmd+Shift+X — otevře se panel Extensions. Do vyhledávacího pole zadej název a klikni Install.

  1. Nainstaluj rozšíření Docker (autor Microsoft) — přidá tlačítka pro ovládání kontejnerů přímo z editoru.
  2. Nainstaluj rozšíření Python (autor Microsoft) — pro správnou práci se soubory Odoo.

Zapnutí automatického formátování: Otevři nastavení zkratkou Cmd+, (čárka). Do vyhledávacího pole napiš format on save a zaškrtni Editor: Format On Save. Díky tomu se YAML soubory při uložení automaticky opraví — špatné odsazení je nejčastější příčina chyb v konfiguračních souborech.

Hotovo, když: V levém panelu Cursoru vidíš ikonku velryby (Docker rozšíření) a Format on Save je zaškrtnuté v nastavení.

Fáze 3 — Konfigurační soubory
5
docker-compose.yml — recept pro Docker
Říká Dockeru co spustit, jak propojit a jak zabezpečit
▶

Co je tento soubor? Textový "recept", který Dockeru říká: spusť backend (PostgreSQL) a frontend (Odoo 18), propoj je dohromady, zpřístupni Odoo na portu 8069, a data ukládej trvale na disk. Bez tohoto souboru Docker neví co má dělat.

Vytvoření souboru:

# Příkaz touch vytvoří prázdný soubor touch docker-compose.yml

V levém panelu Cursoru klikni na docker-compose.yml a vlož celý tento obsah (označ vše Cmd+A, smaž a vlož):

# Zde definujeme všechny "stroje" (services), které Docker spustí services: # ── BACKEND: DATABÁZE ────────────────────────────────────────── # PostgreSQL ukládá všechna data Odoo na disk. # Je to zálohovací systém bez vlastního UI — čistý trezor na data. db: image: postgres:15 # Přesná verze PostgreSQL, kterou Odoo 18 potřebuje environment: - POSTGRES_DB=postgres # Název výchozí databáze - POSTGRES_USER=odoo # Uživatel, přes kterého se Odoo připojuje k DB - POSTGRES_PASSWORD=odoo # Heslo tohoto uživatele (interní, ty ho nikdy nezadáváš) volumes: # Data se ukládají do pojmenovaného svazku. # Díky tomu nepřijdeš o data ani když Docker vypneš. - odoo-db-data:/var/lib/postgresql/data # ── FRONTEND: ODOO ───────────────────────────────────────────── # Odoo je webová aplikace — to co vidíš v prohlížeči. # Čte data z backendu (PostgreSQL) a zobrazuje je jako UI. web: image: odoo:18.0 # Oficiální Odoo 18 image z Docker Hubu depends_on: - db # Odoo počká, až se nejdřív spustí databáze ports: # 127.0.0.1 je loopback adresa — existuje jen uvnitř tvého počítače # a nikdy neopustí síťovou kartu. Nikdo na stejné Wi-Fi se nepřipojí. - "127.0.0.1:8069:8069" volumes: # Propojení složek na tvém počítači se složkami uvnitř kontejneru. - ./config:/etc/odoo # Tvůj odoo.conf → dovnitř kontejneru - ./addons:/mnt/extra-addons # Tvoje vlastní moduly # Odkomentuj (smaž #) až po stažení Enterprise kódu z GitHubu. # Dokud je složka src/ prázdná, Odoo by nespustilo — nenašlo by žádný kód. # - ./src:/usr/lib/python3/dist-packages/odoo/addons environment: - HOST=db # Říká Odoo: databázi najdeš pod názvem "db" (název služby výše) - USER=odoo # Uživatel pro připojení k DB (musí sedět s POSTGRES_USER) - PASSWORD=odoo # Heslo (musí sedět s POSTGRES_PASSWORD) # Pojmenovaný svazek — Docker ho spravuje sám, data přežijí restart volumes: odoo-db-data:

Ulož soubor zkratkou Cmd+S.

Pozor na odsazení: YAML soubory používají mezery (ne tabulátor Tab) pro strukturu — každá úroveň jsou dvě mezery. Díky zapnutému Format on Save se odsazení při uložení automaticky opraví. Pokud Cursor červeně podtrhuje text, je někde chyba v syntaxi.
Hotovo, když: Soubor je uložen, Cursor ho červeně nepodtrhuje.
6
config/odoo.conf — nastavení Odoo aplikace
Musí být ve složce config/, ne v hlavní složce
▶

Co je tento soubor? Říká Odoo kde hledat moduly a jaké je hlavní administrátorské heslo pro správu databází. Musí ležet ve složce config/ — jinak ho Odoo nenajde.

# Vytvoří soubor ve správné podsložce touch config/odoo.conf

V levém panelu klikni na složku config → soubor odoo.conf a vlož:

# Tato hlavička MUSÍ být na prvním řádku — bez ní Odoo celý soubor ignoruje [options] # Cesty kde Odoo hledá moduly — oddělené čárkou, bez mezer # /mnt/extra-addons = tvoje složka addons/ namapovaná z docker-compose.yml # /usr/.../addons = vestavěné moduly přímo v Docker image Odoo addons_path = /mnt/extra-addons,/usr/lib/python3/dist-packages/odoo/addons # Složka pro dočasná data Odoo (session soubory, nahrané přílohy) data_dir = /var/lib/odoo # Master Password — heslo pro správu databází (vytváření, mazání, zálohy) # TOTO NENÍ heslo pro přihlášení do Odoo — to si nastavíš v prohlížeči # Zvol si vlastní silné heslo a zapamatuj si ho admin_passwd = zvol_si_vlastni_heslo

Ulož Cmd+S.

Kriticky důležité: První řádek musí být vždy [options] včetně hranatých závorek. Bez tohoto "nadpisu sekce" Odoo celý soubor ignoruje a budeš vidět chybu v logu.
Hotovo, když: Soubor config/odoo.conf existuje, začíná [options] a obsahuje tvé vlastní admin_passwd.
Fáze 4 — Přehled hesel
7
Tři různá hesla — která jsou kde a k čemu
Nepomíchej je — každé odemyká něco jiného
▶

V systému jsou tři hesla na třech různých úrovních. Je zásadní vědět, co každé dělá — záměna způsobí chyby, které jsou těžko dohledatelné.

Interní
DB heslo
docker-compose.yml
Technické spojení backendu s frontendem. Slouží výhradně pro komunikaci mezi Odoo a PostgreSQL uvnitř Dockeru. Jako uživatel ho nikdy nezadáváš — funguje automaticky na pozadí. Pro lokální vývoj je "odoo/odoo" bezpečné.
Master
admin_passwd
config/odoo.conf
Klíč od trezoru serveru. Používáš ho jen na stránce Database Manager — pro vytvoření, smazání nebo zálohu celé databáze. Zadáváš ho jednorázově při prvním spuštění. Zvol silné heslo.
Uživatel
Tvůj login
webový formulář
Každodenní přihlášení do Odoo. Nastavíš si ho při inicializaci databáze v prohlížeči. Tímto heslem se přihlašuješ do Odoo každý den.
Jednou větou: DB heslo = propojení programů (automatické, nevidíš ho). Master heslo = správa serverů (jednorázové). Uživatelské heslo = tvůj každodenní přístup.
Fáze 5 — Spuštění
8
Spuštění kontejnerů
Docker stáhne Odoo a databázi a spustí je
▶

Před spuštěním ověř, že Docker běží: Otevři terminál v Cursoru (Ctrl+`) a zadej:

# Ověření že Docker je spuštěný a terminál ho vidí docker --version
Docker version 27.3.1, build ce12230

Pokud terminál napíše "command not found", Docker Desktop není spuštěný — otevři ho z Launchpadu a počkej na ikonku velryby v systémové liště macOS.

Ujisti se, že jsi ve správné složce — v terminálu to vidíš v textu před kurzorem (např. princparket18 %). Pokud ne:

cd ~/Desktop/princparket18

Spuštění:

# -d = detached, kontejnery běží na pozadí a terminál zůstane volný docker compose up -d

Docker stáhne obrazy PostgreSQL 15 a Odoo 18.0 (řádky s textem "Pulling"). První stažení trvá 5–15 minut.

Správný výstup po spuštění:

✔ Container princparket18-db-1 Started ✔ Container princparket18-web-1 Started

Ověření stavu:

# Zobrazí stav všech kontejnerů — u obou musí být "running" docker compose ps

Zobrazení logů (co Odoo dělá uvnitř, kde jsou chyby):

# -f = sleduj průběžně. Ukončíš Ctrl+C — Odoo tím nevypneš docker compose logs -f web
Nejčastější chyba — "command not found": Docker Desktop není spuštěný. Otevři ho z Launchpadu.
Pozor na pomlčku: Správně je docker compose (mezera). Starý příkaz docker-compose s pomlčkou v nových verzích nefunguje.
Hotovo, když: Příkaz "docker compose ps" zobrazuje u obou služeb status "running".
Fáze 6 — Odoo v prohlížeči
9
Inicializace databáze v prohlížeči
Jednorázové nastavení — vytvoříš zde svůj systém
▶

Otevři Chrome (doporučujeme Chrome — nejlepší kompatibilita s Odoo) a zadej do adresního řádku:

http://localhost:8069

Pokud se stránka nenačte hned, počkej 30 sekund a obnov (Cmd+R). Odoo potřebuje chvíli na první start.

Vyplň formulář "Create Database":

  1. Master Password — heslo z config/odoo.conf (to co jsi napsal za admin_passwd)
  2. Database Name — např. princparket_test. Jen písmena, číslice a podtržítko — bez mezer a háčků.
  3. Email — tvůj přihlašovací e-mail do Odoo (nemusí existovat, slouží jako login)
  4. Password — heslo pro každodenní přihlašování do Odoo
  5. Demo Data — viz varování níže
Demo Data — nevratná akce, dobře si to rozmysli Zaškrtnutím přidáš do databáze vzorová data — fiktivní zákazníky, faktury, produkty. Tato data nelze hromadně smazat bez smazání celé databáze. Zaškrtni pokud: Chceš si Odoo proklikat a vyzkoušet jak vypadá vyplněný systém. Ideální pro první seznámení. Nezaškrtni pokud: Budeš databázi používat pro reálný provoz nebo do ní importovat data klienta — demo záznamy by se mísily s reálnými.

Klikni na Create Database a počkej 1–3 minuty.

Vidíš "Community" místo "Enterprise"? To je v pořádku — Enterprise moduly přidáme po stažení kódu z GitHubu. Základní funkce fungují.
Stránka se nenačítá: Zkontroluj v terminálu příkazem docker compose ps jestli kontejnery běží. Pokud je status "exited", spusť docker compose logs web a pošli chybovou hlášku.
Hotovo, když: Vidíš přihlašovací obrazovku nebo dashboard Odoo. Lokální prostředí běží!
Cheat sheet

Orientace v terminálu — základní příkazy
Jak se pohybovat v adresářích, co příkazy dělají a jak zapnout skryté soubory
▶

Terminál vždy pracuje v nějakém adresáři (složce). Aktuální adresář vidíš v textu před kurzorem, obvykle za vlnovkou ~. Cesta se skládá ze složek oddělených lomítkem, např. /Users/jan/Desktop/princparket18.

PříkazCo dělá
pwdPrint Working Directory — zobrazí celou cestu k aktuálnímu adresáři. Používej kdykoli si nejsi jistý, kde se nacházíš.
lsVypíše obsah aktuálního adresáře — soubory a složky. Skryté soubory (začínají tečkou) nezobrazí.
ls -laVypíše obsah včetně skrytých souborů (začínají tečkou, např. .env) a detailů jako velikost a datum. Soubory začínající tečkou jsou záměrně skryté — systémové konfigurace, které by měly být méně viditelné. Bez parametru -a je Finder ani terminál nezobrazí.
cd názevVstoupí do podsložky. Např. cd config přejde do složky config uvnitř aktuálního adresáře.
cd ..Přejde o úroveň výš (do nadřazené složky). Dvě tečky = rodič.
cd ~Přejde do domovské složky (/Users/tvoje_jméno) — vždy funguje odkudkoliv.
cd ~/DesktopPřejde přímo na Plochu. Tilda (~) je zkratka pro domovskou složku.
mkdir názevMake Directory — vytvoří novou složku v aktuálním adresáři.
touch souborVytvoří prázdný soubor. Obsah pak přidáš v editoru (Cursoru).
clearVyčistí terminál — starý výstup zmizí, nic se nesmaže ani nevypne.
Ctrl+CPřeruší běžící příkaz (např. sledování logů). Nic nesmaže ani nevypne.

Jak zapnout skryté soubory ve Finderu: Stiskni Cmd+Shift+. (tečka) v libovolném okně Finderu. Skryté soubory se zobrazí šedě. Stejnou zkratkou je opět skryješ. Skryté soubory v projektu (jako .env s hesly) jsou záměrně skryté, abys je náhodně nenahrál na GitHub.

Každodenní příkazy pro Odoo:

# Spuštění Odoo (po restartu počítače nebo po "down") docker compose up -d # Zastavení Odoo — BEZPEČNÉ, data zůstanou zachována docker compose down # Zobrazení stavu kontejnerů docker compose ps # Sledování logů Odoo v reálném čase (ukončit: Ctrl+C) docker compose logs -f web # Restart po změně konfiguračních souborů docker compose down && docker compose up -d
Toto smaže všechna data — používej jen záměrně: docker compose down -v — parametr -v smaže i datové svazky včetně celé databáze. Bez parametru -v jsou data v bezpečí.

Fáze 0 — Příprava Windows
0
WSL2 — podsystém pro Linux (nutný základ)
Windows potřebuje tuto vrstvu, aby Docker fungoval
▶

Co je WSL2? Docker na Windows potřebuje ke svému běhu linuxové prostředí. WSL2 (Windows Subsystem for Linux) je tenká linuxová vrstva zabudovaná přímo do Windows. Bez ní Docker nefunguje.

Instalace WSL2: Klikni pravým tlačítkem na tlačítko Start a vyber "Windows PowerShell (admin)" nebo "Terminál (admin)". Zadej příkaz:

# Nainstaluje WSL2 i výchozí Linux distribuci (Ubuntu) wsl --install

Po dokončení restartuj počítač.

Windows 10 vs 11: Na Windows 10 musíš mít verzi 2004 nebo novější (Build 19041+). Zkontroluj přes Nastavení → Systém → O aplikaci Windows.
Hotovo, když: Počítač se restartoval a příkaz wsl --status v PowerShellu zobrazí "Default Version: 2".

Fáze 3 — Konfigurační soubory
5
docker-compose.yml — recept pro Docker
Říká Dockeru co spustit, jak propojit a jak zabezpečit
▶

Co je tento soubor? Textový "recept", který Dockeru říká: spusť backend (PostgreSQL) a frontend (Odoo 18), propoj je dohromady, zpřístupni Odoo na portu 8069, a data ukládej trvale na disk. Bez tohoto souboru Docker neví co má dělat.

Soubor jsi vytvořil v kroku 4 příkazem New-Item docker-compose.yml. Teď ho naplníme obsahem.

V levém panelu Cursoru klikni na docker-compose.yml a vlož celý tento obsah (označ vše Ctrl+A, smaž a vlož):

# Zde definujeme všechny "stroje" (services), které Docker spustí services: # ── BACKEND: DATABÁZE ────────────────────────────────────────── # PostgreSQL ukládá všechna data Odoo na disk. # Je to zálohovací systém bez vlastního UI — čistý trezor na data. db: image: postgres:15 # Přesná verze PostgreSQL, kterou Odoo 18 potřebuje environment: - POSTGRES_DB=postgres # Název výchozí databáze - POSTGRES_USER=odoo # Uživatel, přes kterého se Odoo připojuje k DB - POSTGRES_PASSWORD=odoo # Heslo tohoto uživatele (interní, ty ho nikdy nezadáváš) volumes: # Data se ukládají do pojmenovaného svazku. # Díky tomu nepřijdeš o data ani když Docker vypneš. - odoo-db-data:/var/lib/postgresql/data # ── FRONTEND: ODOO ───────────────────────────────────────────── # Odoo je webová aplikace — to co vidíš v prohlížeči. # Čte data z backendu (PostgreSQL) a zobrazuje je jako UI. web: image: odoo:18.0 # Oficiální Odoo 18 image z Docker Hubu depends_on: - db # Odoo počká, až se nejdřív spustí databáze ports: # 127.0.0.1 je loopback adresa — existuje jen uvnitř tvého počítače # a nikdy neopustí síťovou kartu. Nikdo na stejné Wi-Fi se nepřipojí. - "127.0.0.1:8069:8069" volumes: # Propojení složek na tvém počítači se složkami uvnitř kontejneru. - ./config:/etc/odoo # Tvůj odoo.conf → dovnitř kontejneru - ./addons:/mnt/extra-addons # Tvoje vlastní moduly # Odkomentuj (smaž #) až po stažení Enterprise kódu z GitHubu. # Dokud je složka src/ prázdná, Odoo by nespustilo — nenašlo by žádný kód. # - ./src:/usr/lib/python3/dist-packages/odoo/addons environment: - HOST=db # Říká Odoo: databázi najdeš pod názvem "db" (název služby výše) - USER=odoo # Uživatel pro připojení k DB (musí sedět s POSTGRES_USER) - PASSWORD=odoo # Heslo (musí sedět s POSTGRES_PASSWORD) # Pojmenovaný svazek — Docker ho spravuje sám, data přežijí restart volumes: odoo-db-data:

Ulož soubor zkratkou Ctrl+S.

Pozor na odsazení: YAML soubory používají mezery (ne tabulátor Tab) pro strukturu — každá úroveň jsou dvě mezery. Díky zapnutému Format on Save se odsazení při uložení automaticky opraví. Pokud Cursor červeně podtrhuje text, je někde chyba v syntaxi.
Hotovo, když: Soubor je uložen, Cursor ho červeně nepodtrhuje.
6
config/odoo.conf — nastavení Odoo aplikace
Musí být ve složce config/, ne v hlavní složce
▶

Co je tento soubor? Říká Odoo kde hledat moduly a jaké je hlavní administrátorské heslo pro správu databází. Musí ležet ve složce config/ — jinak ho Odoo nenajde.

Soubor jsi vytvořil v kroku 4 příkazem New-Item config\odoo.conf. Teď ho naplníme obsahem.

V levém panelu klikni na složku config → soubor odoo.conf a vlož:

# Tato hlavička MUSÍ být na prvním řádku — bez ní Odoo celý soubor ignoruje [options] # Cesty kde Odoo hledá moduly — oddělené čárkou, bez mezer # /mnt/extra-addons = tvoje složka addons/ namapovaná z docker-compose.yml # /usr/.../addons = vestavěné moduly přímo v Docker image Odoo addons_path = /mnt/extra-addons,/usr/lib/python3/dist-packages/odoo/addons # Složka pro dočasná data Odoo (session soubory, nahrané přílohy) data_dir = /var/lib/odoo # Master Password — heslo pro správu databází (vytváření, mazání, zálohy) # TOTO NENÍ heslo pro přihlášení do Odoo — to si nastavíš v prohlížeči # Zvol si vlastní silné heslo a zapamatuj si ho admin_passwd = zvol_si_vlastni_heslo

Ulož Ctrl+S.

Kriticky důležité: První řádek musí být vždy [options] včetně hranatých závorek. Bez tohoto "nadpisu sekce" Odoo celý soubor ignoruje a budeš vidět chybu v logu.
Hotovo, když: Soubor config/odoo.conf existuje, začíná [options] a obsahuje tvé vlastní admin_passwd.
Fáze 4 — Přehled hesel
7
Tři různá hesla — která jsou kde a k čemu
Nepomíchej je — každé odemyká něco jiného
▶

V systému jsou tři hesla na třech různých úrovních. Je zásadní vědět, co každé dělá — záměna způsobí chyby, které jsou těžko dohledatelné.

Interní
DB heslo
docker-compose.yml
Technické spojení backendu s frontendem. Slouží výhradně pro komunikaci mezi Odoo a PostgreSQL uvnitř Dockeru. Jako uživatel ho nikdy nezadáváš — funguje automaticky na pozadí. Pro lokální vývoj je "odoo/odoo" bezpečné.
Master
admin_passwd
config/odoo.conf
Klíč od trezoru serveru. Používáš ho jen na stránce Database Manager — pro vytvoření, smazání nebo zálohu celé databáze. Zadáváš ho jednorázově při prvním spuštění. Zvol silné heslo.
Uživatel
Tvůj login
webový formulář
Každodenní přihlášení do Odoo. Nastavíš si ho při inicializaci databáze v prohlížeči. Tímto heslem se přihlašuješ do Odoo každý den.
Jednou větou: DB heslo = propojení programů (automatické, nevidíš ho). Master heslo = správa serverů (jednorázové). Uživatelské heslo = tvůj každodenní přístup.
Fáze 5 — Spuštění
8
Spuštění kontejnerů
Docker stáhne Odoo a databázi a spustí je
▶

Před spuštěním ověř, že Docker běží: Otevři terminál v Cursoru (Ctrl+`) a zadej:

# Ověření že Docker je spuštěný a terminál ho vidí docker --version
Docker version 27.3.1, build ce12230

Pokud terminál napíše "Tento příkaz nebyl rozpoznán", Docker Desktop není spuštěný — otevři ho z nabídky Start a počkej na ikonku velryby v oznamovací oblasti.

Ujisti se, že jsi ve správné složce — v terminálu to vidíš v textu před kurzorem (např. PS C:\Users\Jan\Desktop\princparket18>). Pokud ne:

cd $HOME\Desktop\princparket18

Spuštění:

# -d = detached, kontejnery běží na pozadí a terminál zůstane volný docker compose up -d

Docker stáhne obrazy PostgreSQL 15 a Odoo 18.0 (řádky s textem "Pulling"). První stažení trvá 5–15 minut.

Správný výstup po spuštění:

✔ Container princparket18-db-1 Started ✔ Container princparket18-web-1 Started

Ověření stavu:

# Zobrazí stav všech kontejnerů — u obou musí být "running" docker compose ps

Zobrazení logů (co Odoo dělá uvnitř, kde jsou chyby):

# -f = sleduj průběžně. Ukončíš Ctrl+C — Odoo tím nevypneš docker compose logs -f web
Nejčastější chyba — "příkaz nebyl rozpoznán": Docker Desktop není spuštěný. Otevři ho z nabídky Start.
Pozor na pomlčku: Správně je docker compose (mezera). Starý příkaz docker-compose s pomlčkou v nových verzích nefunguje.
Hotovo, když: Příkaz "docker compose ps" zobrazuje u obou služeb status "running".
Fáze 6 — Odoo v prohlížeči
9
Inicializace databáze v prohlížeči
Jednorázové nastavení — vytvoříš zde svůj systém
▶

Otevři Chrome (doporučujeme Chrome — nejlepší kompatibilita s Odoo) a zadej do adresního řádku:

http://localhost:8069

Pokud se stránka nenačte hned, počkej 30 sekund a obnov stránku (F5). Odoo potřebuje chvíli na první start.

Uvidíš stránku "Create Database" — vyplň ji takto:

  1. Master Password — heslo z config/odoo.conf (to co jsi napsal za admin_passwd)
  2. Database Name — název tvé databáze, např. princparket_test. Jen písmena, číslice a podtržítko — bez mezer a háčků.
  3. Email — tvůj přihlašovací e-mail do Odoo (nemusí existovat, slouží jako login)
  4. Password — heslo pro každodenní přihlašování do Odoo
  5. Demo Data — viz varování níže
Demo Data — nevratná akce, dobře si to rozmysli Zaškrtnutím přidáš do databáze vzorová data — fiktivní zákazníky, faktury, produkty. Tato data nelze hromadně smazat bez smazání celé databáze. Zaškrtni pokud: Chceš si Odoo proklikat a vyzkoušet jak vypadá vyplněný systém. Ideální pro první seznámení. Nezaškrtni pokud: Budeš databázi používat pro reálný provoz nebo do ní importovat data klienta — demo záznamy by se mísily s reálnými.

Klikni na Create Database a počkej 1–3 minuty.

Vidíš "Community" místo "Enterprise"? To je v pořádku — Enterprise moduly přidáme po stažení kódu z GitHubu. Základní funkce fungují.
Stránka se nenačítá: Zkontroluj v terminálu příkazem docker compose ps jestli kontejnery běží. Pokud je status "exited", spusť docker compose logs web a pošli chybovou hlášku.
Hotovo, když: Vidíš přihlašovací obrazovku nebo dashboard Odoo. Lokální prostředí běží!
Cheat sheet
T
Orientace v terminálu — základní příkazy (PowerShell)
Jak se pohybovat v adresářích, co příkazy dělají a jak zapnout skryté soubory
▶

PowerShell vždy pracuje v nějakém adresáři. Aktuální adresář vidíš v textu před kurzorem, např. PS C:\Users\Jan\Desktop\princparket18>. Cesta se skládá ze složek oddělených zpětným lomítkem \.

PříkazCo dělá
pwdPrint Working Directory — zobrazí celou cestu k aktuálnímu adresáři. Používej kdykoli si nejsi jistý, kde se nacházíš.
ls nebo dirVypíše obsah aktuálního adresáře — soubory a složky. Obě varianty fungují v PowerShellu stejně. Skryté soubory (začínají tečkou) zobrazí také.
ls -ForceZobrazí všechny soubory včetně skrytých — soubory s atributem Hidden, které by jinak nebyly vidět. Ekvivalent macOS ls -la.
cd názevVstoupí do podsložky. Např. cd config přejde do složky config uvnitř aktuálního adresáře.
cd ..Přejde o úroveň výš (do nadřazené složky).
cd $HOMEPřejde do domovské složky (C:\Users\tvoje_jméno).
cd $HOME\DesktopPřejde přímo na Plochu.
mkdir názevMake Directory — vytvoří novou složku. Funguje stejně jako na macOS.
New-Item souborVytvoří prázdný soubor (Windows ekvivalent macOS příkazu touch).
clsVyčistí terminál — starý výstup zmizí, nic se nesmaže.
Ctrl+CPřeruší běžící příkaz. Nic nesmaže ani nevypne.

Jak zapnout skryté soubory ve Windows Průzkumníku: Otevři libovolné okno Průzkumníku → záložka Zobrazení (nebo View) → zaškrtni Skryté položky. Případně přes Nastavení → Průzkumník → Zobrazit skryté soubory. Soubory začínající tečkou (jako .env s hesly) jsou záměrně skryté, abys je náhodně nenahrál na GitHub.

Každodenní příkazy pro Odoo (stejné jako na macOS):

# Spuštění Odoo docker compose up -d # Zastavení Odoo — BEZPEČNÉ, data zůstanou zachována docker compose down # Zobrazení stavu kontejnerů docker compose ps # Sledování logů Odoo v reálném čase (ukončit: Ctrl+C) docker compose logs -f web # Restart po změně konfiguračních souborů docker compose down && docker compose up -d
Toto smaže všechna data — používej jen záměrně: docker compose down -v — parametr -v smaže i datové svazky včetně celé databáze. Bez parametru -v jsou data v bezpečí.
